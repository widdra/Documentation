# nginx
HTTP and reverse proxy server
<br/>
[Homepage](//nginx.org/) |
[Documentation](//nginx.org/en/docs/)

## Installation
``` bash
sudo apt-get update
sudo apt-get install nginx
sudo ufw allow 'Nginx Full'
```

## Directory structure
``` text
/etc/nginx
├── conf.d/                        # configuration files to include on all sites
├── include.d/                     # configuration files to manually include
│   └── ssl.conf
├── nginx.conf                     # global configuration file
├── sites-available/               # site configuration files
│   ├── aspnetcore
│   ├── default
│   ├── httpredirect
│   └── nodejs
└── sites-enabled/                 # symlinks to sites-available to enable site
    ├── aspnetcore -> /etc/nginx/sites-available/aspnetcore
    ├── httpredirect -> /etc/nginx/sites-available/httpredirect
    └── nodejs -> /etc/nginx/sites-available/nodejs
    
/var/www/html                      # default web content directory
└── index.nginx-debian.html
```

## Main Configuration File
nginx configuration file (`/etc/nginx/nginx.conf`):
``` text
user www-data;
worker_processes 1;
pid /run/nginx.pid;

events {
        worker_connections 1024;
}

http {
  include /etc/nginx/proxy.conf;

        sendfile on;
        tcp_nopush on;
        tcp_nodelay on;
        keepalive_timeout 65;
        types_hash_max_size 2048;

        include /etc/nginx/mime.types;
        default_type application/octet-stream;

        ssl_protocols TLSv1 TLSv1.1 TLSv1.2; # Dropping SSLv3, ref: POODLE
        ssl_prefer_server_ciphers on;

        access_log /var/log/nginx/access.log;
        error_log /var/log/nginx/error.log;

        gzip on;
        gzip_disable "msie6";
        # gzip_vary on;
        # gzip_proxied any;
        # gzip_comp_level 6;
        # gzip_buffers 16 8k;
        # gzip_http_version 1.1;
        # gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

        include /etc/nginx/conf.d/*.conf;
        include /etc/nginx/sites-enabled/*;
}
```

Redirect http to https (`/etc/nginx/sites-available/httpredirect`):
``` text
server {
        listen *:80;
        add_header Strict-Transport-Security max-age=15768000;
        return 301 https://$host$request_uri;
}
```

Simple reverse proxy (`/etc/nginx/sites-available/aspnetcore`):
``` text
server {
        listen                      *:443 ssl;
        server_name                 api.example.de;
        include                     /etc/nginx/include.d/ssl.conf;
        root                        /var/www/public;
        location ~ /.well-known {
              allow                     all;
        }
        location / {
              proxy_pass                http://127.0.0.1:5000/;
        }
}
```

Reverse proxy with some static content (`/etc/nginx/sites-available/nodejs`):
```
server {
        listen                      *:443 ssl;
        server_name                 example.de www.example.de;
        include                     /etc/nginx/include.d/ssl.conf;
        root                        /var/www/public;

        location ~ /.well-known {
              allow                 all;
        }
        location /license {
              alias                 /var/www/public/license;
        }
        location /images {
              alias                 /var/www/static/images;
              autoindex             on;
        }
        location / {
              proxy_pass            http://127.0.0.1:5050/;
        }
}
```

## Service Management
To restart the whole nginx process and all worker processes use `sudo systemctl restart nginx`.
<br/>
In most cases it is not neccessary to restart the whole service to reload changed configuration files.
<br/>
Just use `sudo systemctl reload nginx` instead.
